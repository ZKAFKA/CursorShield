企业级 Cursor 安全监控插件技术方案
1. 项目目标
开发一款不可绕过的企业安全监控插件，覆盖以下核心能力：

① 登录账号监控：读取当前 Cursor 登录邮箱，确保仅允许企业账号使用。

② Git 仓库审计：实时监控项目关联的 GitHub 地址，阻断非授权仓库推送。

③ MCP / Skill 扫描：发现并审计项目内加载的所有 AI 扩展能力。

⑤ 敏感信息检测：在保存、暂存、提交、剪贴板等多环节拦截密钥、令牌等泄露。

防绕过机制：确保插件始终运行，防止开发者禁用、卸载或篡改配置。

3. 功能详细设计
3.1 登录账号监控
实现方式：

读取 Cursor 本地数据库 state.vscdb，查询 cursorAuth/cachedEmail。

与管理员配置的 allowedDomains（如 mycompany.com）比对。

状态栏显示邮箱，合规为 ✅，否则显示 ⚠️ 并高亮。

跨平台数据库路径：

操作系统	路径
macOS	~/Library/Application Support/Cursor/User/globalStorage/state.vscdb
Windows	%APPDATA%\Cursor\User\globalStorage\state.vscdb
Linux	~/.config/Cursor/User/globalStorage/state.vscdb
安全约束：仅本地读取，不修改任何数据，不通过网络传输。

3.2 Git 仓库监控
功能点：

获取 origin 远程地址（fetchUrl）。

与配置项 allowedRepos（支持正则列表）比对。

仓库切换时实时校验；不匹配时状态栏告警，并可阻断 git.push 操作。

核心代码思路：

typescript
const git = vscode.extensions.getExtension('vscode.git')?.exports.getAPI(1);
const repo = git.repositories[0];
const remote = repo.state.remotes.find(r => r.name === 'origin');
const url = remote?.fetchUrl;
// 检查 url 是否匹配 allowedRepos
阻断非授权推送：拦截 git.push 命令，比对目标远程地址。

3.3 MCP / Skill 扫描
MCP 检测：

读取 .cursor/mcp.json，解析 mcpServers 字段。

列出所有注册的 server 名称、命令及环境变量（脱敏显示）。

Skill 检测：

遍历 .cursor/skills/ 下的子目录，每个子目录为一个 Skill。

读取 skill.json 获取名称、描述。

合规管控：

提供 blockedMCPs / allowedSkills 配置项。

发现未授权项时在仪表盘标红，并可弹窗提示安全风险。

展示示例：

text
🧩 MCP: figma (⚠️未授权), postgres (✅)
⚡ Skills: react-patterns (✅), python-security (✅)
3.5 敏感信息检测（核心）
3.5.1 检测卡点
触发时机	监听方式	作用
文件保存	onDidSaveTextDocument	防止明文写入密钥
添加到暂存区	拦截 git.stage / git.add	阻止敏感信息进入 Git
提交前	拦截 git.commit	最后一道防线
剪贴板复制	vscode.env.clipboard.onDidWriteText	防止密钥通过剪切板带出 IDE
编辑器实时装饰器	onDidChangeTextDocument	输入时即标记疑似密钥
对于 AI 输出内容，如 Cursor Chat 面板的虚拟文档，可通过监听 onDidChangeTextDocument 并过滤 scheme === 'vscode-chat'（若可用）来检测，否则依赖剪切板监控覆盖。

3.5.2 规则引擎
内置规则库部分示例：

类别	正则表达式
AWS Access Key	AKIA[0-9A-Z]{16}
GitHub Personal Token	ghp_[0-9a-zA-Z]{36}
GitLab Token	glpat-[0-9a-zA-Z\-]{20,}
Slack Token	xox[baprs]-[0-9a-zA-Z\-]{10,}
私钥	-----BEGIN (RSA|DSA|EC|OPENSSH) PRIVATE KEY-----
数据库连接串	(jdbc|mongodb|mysql|postgresql)://[^:]+:[^@]+@
通用高熵字符串	[A-Za-z0-9+/]{40,}（需配合熵值阈值，默认 >4.5）
自定义规则：支持用户在 .cursor-security-rules.toml 中添加自定义正则。

3.5.4 处理流程（以 Git Commit 为例）
用户执行 git.commit。

插件拦截，获取暂存区变更文件列表。

对每个文件的新增内容进行扫描（正则 + 熵检测）。

发现匹配项 → 弹出错误框，显示文件路径、行号、匹配类型（部分脱敏）。

阻断提交，直到所有问题被清除。

记录审计日志（时间、账号、仓库、匹配规则、处理结果）。

4. 防绕过机制
防护层级	实现方案
禁用监控	监听 vscode.extensions.onDidChange，当自身状态变为禁用时，立刻弹出严重警告，并尝试通过命令行重新启用（需管理员权限）；心跳上报异常状态。
卸载阻断	通过企业设备管理策略（MDM / Group Policy）强制安装并锁定；同时监控自身扩展目录，若文件被删除则告警。
配置防篡改	监听 vscode.workspace.onDidChangeConfiguration，若安全策略被改为无效值，立即恢复为上一次可信配置。
心跳机制	每隔 5 分钟向内部安全服务发送心跳（包含扩展状态、账号、仓库等）。服务端若未收到心跳，可发出告警。
文件完整性	使用 fs.watch 监控自身扩展根目录，检测脚本被替换或删除时触发紧急告警。
建议结合企业终端管理软件（如 Jamf、Intune）将插件设为强制安装且禁止卸载，实现最强的不可绕过性。

8. 部署与分发
打包：npx vsce package 生成 .vsix。

分发：

内部扩展市场。

通过 MDM / Group Policy 强制推送并锁定。

更新：插件支持自动更新，但更新后需验证策略完整性。

9. 注意事项与风险
数据库路径变化：Cursor 更新可能移动 state.vscdb 位置，需定期适配。

性能：大文件熵计算可能耗时，建议只扫描新增或变更行，并设置最大扫描行数（默认 5000 行）。

隐私：读取的账号、仓库等仅做本地校验，不上报（除非开启心跳功能）。

合规：确保企业策略强制使用本插件，避免个人开发者绕过。
